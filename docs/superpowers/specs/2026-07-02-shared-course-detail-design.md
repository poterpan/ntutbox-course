# 分享課表 — 點課看詳情 + 收藏/排入（設計）

- 日期：2026-07-02
- 對應 issue：#32（F-B 加分）
- 前置：#31（F-B 分享整份課表，已 merge）

## 需求（釐清後）

學生收到別人分享的課表時，**合併/取代整份反而最少用**——大家課表不會一樣、各班必修不同，很少人整份照抄。真正的行為是：**看同學修了哪些選修課 → 有興趣的收起來 → 之後自己考慮要不要加選**。

因此核心動作是「**加入收藏**」（存起來、之後考慮加選），並且「**排入單堂**」也開放——既然整份都能匯入自己的排課，沒理由擋單堂加選。

現況：分享彈窗（`SharedTimetableModal`）裡的課程**只能看、不能點**，也不能收藏/排入。主排課器裡課程本來就能點開詳情（`CourseDetailDrawer`）並收藏/排入，只有這個彈窗點不動 → 是個不一致，也擋住了上面的真實需求。

## 目標

分享彈窗裡點任一門課 → **就地展開**該課詳情，並可 **收藏** 或 **排入**。

## 非目標（YAGNI）

- **不**在週課表格子上放「快速收藏 ★」（會讓課表變亂；使用者明確不要）。
- **不**在本次重做/移除「合併/取代整份」——先保留不動；要不要弱化它另開議題。
- **不**新增「分享課表 vs 我的課表」比較視圖。

## 互動流程

1. 分享彈窗預設顯示週課表 + 課程清單 + 匯入區（現況）。
2. 點清單列（主要）或週課表格子（順手，同一 handler）→ 彈窗內容**原地**切成該課詳情。
3. 詳情頂部有 **← 返回課表**；底部有 **☆收藏／★已收藏** ＋ **＋排入課表／退選**，並顯示當前狀態。
4. 收藏：toggle，停留在詳情（可繼續看、再排入）。
5. 排入：加進使用者草稿（與課程庫排入完全一致）→ 回到課表清單 + toast。
6. 返回 → 回到週課表清單，可再點下一門。

## 架構

**不開新的 dialog**（避免彈窗疊彈窗）。改為在分享彈窗**同一個 `DialogContent` 內切換 view**（master↔detail）。

- 抽出共用元件 **`CourseDetailContent`**（放 `components/planner/`）：課程詳情資訊 body（師資/時間/教室/概述/大綱 + 載入/空狀態）＋ 動作 footer（收藏 + 排入/退選）＋ 分享。用 props 參數化差異：
  - `offeringId: string`
  - `onAfterPlace?: () => void` — 排入/退選後的動作（抽屜：`openDetail(null)` 關閉；分享彈窗：回到清單 view）
  - 需要時傳入 scroll ref / initialFocus，或元件內自理
- **`CourseDetailDrawer`（現有）**：改成外層仍是自己的 `Dialog`，內容改用 `CourseDetailContent`。**對外行為完全不變**。
- **`SharedTimetableModal`（新）**：加 local state `detailId: string | null`。
  - `detailId == null` → 現況（grid + list + 匯入區）。
  - `detailId != null` → 顯示 `CourseDetailContent`（含 ← 返回 = `setDetailId(null)`）。
  - 清單列 / 格子的 onClick → `setDetailId(offering_id)`。

狀態一律重用 `draft-store`（`favorites` / `placed` / `toggleFavorite` / `place` / `unplace`），所以「已收藏/已排」在兩處與課程庫自動一致。

## 邊界 / 錯誤

- 已下架課號（`byId` 查不到）本來就被 `validIds` 濾掉、不會出現在清單/格子、不可點 → 不需額外處理。
- 排入的衝堂偵測沿用現有 draft 邏輯（排入後在主排課器照常顯示衝堂）；詳情本身不特別標「跟你課表衝堂」（與現有抽屜一致）。
- 排入單堂後，分享彈窗的清單仍讀 `sharedPlan`（不受影響）。若之後又按整份「取代」會清空草稿（含剛排入的）——此為兩路徑並存的已知取捨，`取代` 文案已明示「會清空目前排課」。

## 主要風險

抽 `CourseDetailContent` 會動到**目前正常運作**的 `CourseDetailDrawer`。抽完必須確認抽屜行為**完全不變**：課程詳情載入 effect、觸控捲動聚焦（`useTouchScrollFocus`）、分享、收藏/排入/退選狀態切換。→ 先抽、先驗抽屜沒壞，再接分享彈窗。
